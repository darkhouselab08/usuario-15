# specs/ — Especificaciones técnicas

Specs aprobados antes de ejecutar tareas de código. Flujo Vertex Pather:
**redactar → Franco aprueba → ejecutar.**

## Convención

Archivo: `NN_nombre_spec.md` (ej. `01_externalizar_css_js.md`).

Cada spec incluye: objetivo, alcance, criterios de aceptación y plan de verificación.

## Estado

| Spec | Título | Estado |
|------|--------|--------|
| — | (ninguno aún) | — |

Al aprobar un spec, se ejecuta en una branch `feature/NN-descripcion`.
